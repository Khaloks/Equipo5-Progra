Buenas Prácticas en la Industria para el uso de git y commit

1. Buenas prácticas de GitHub

Git tiene un valor importante en la industria ya que es una manera en la que se puede trabajar colaborativamente. Hay ciertas maneras en las que una persona puede obtener mejores resultados utilizando estas plataformas.

Para lograr ser más eficiente, es recomendado hacer “commits” limpios y con un solo propósito. Esto ayuda a organizar las correcciones o adiciones futuras para que todo quede claro para el equipo y evitar hacer un cambio muy drástico en el proceso. De la misma manera, crear commits seguidos en lugar de uno largo para así asegurar que todo se queda guardado y actualizado. También probar el código antes de confirmarlo puede ser útil para solamente tener "commits" finales, funcionales y completos. Para mejorar la comunicación entre el equipo, se debe especificar de manera imperativa lo que cada "commit" está logrando. Es importante que una vez que el "commit" quede listo, se le agregue un nombre claro y relevante para lograr identificar el cambio rápidamente.

Es recomendado no alterar cualquier modificación que ya haya sido publicada ya que puede resultar en problemas para el resto del equipo con acceso al repositorio. De igual manera, una vez que se genera un archivo, es importante no confirmarlo al menos que haya sido creado manualmente.
